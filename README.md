# Store Database Schema

A complete relational database design for order and inventory management systems.



### What This Project Does

This project provides a fully normalized database schema for managing a retail or wholesale business. It handles products, customer orders, inventory tracking, supplier relationships, shipping logistics, and employee management in an integrated system.



### Why This Project Is Useful

Whether you're building an e-commerce platform, a wholesale distribution system, or learning database design, this schema provides a production-ready foundation. It demonstrates proper normalization, foreign key relationships, and real-world business logic including order processing workflows, inventory management, and hierarchical employee structures.



### Getting Started

The database includes these core tables: Products, Orders, Order\_Details, Customers, Employees, Suppliers, Categories, and Shippers.

To use this schema:

1. Review the ERD diagram to understand table relationships
2. Execute the SQL scripts in your preferred database system (MySQL, PostgreSQL, SQL Server, etc.)
3. Adapt field types and constraints to match your specific database platform
4. Populate with sample data or integrate with your application



### Key Features

* Complete order-to-cash workflow support
* Inventory tracking with reorder levels
* Customer and supplier management
* Employee hierarchy with reporting structure
* Multi-item orders with line-level pricing and discounts
* Shipping integration with multiple carriers
